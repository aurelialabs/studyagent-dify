# 面试讲解稿

## 30 秒介绍

我做了一个叫考途 StudyAgent 的教育 Agent，面向考研、考公和课程考试等教材复习场景。它解决的是学生面对教材时重点难判断、笔记整理耗时、复习计划不具体的问题。用户输入教材章节内容、考试类型、当前基础和每日学习时间后，系统会生成章节结构、重点知识、背诵笔记、自测题、答案解析和复习计划。我用 Dify Workflow 将任务拆成多个 LLM 节点，并通过结构化 Prompt 提升输出稳定性。

## 2 分钟介绍

这个项目来源于备考学生的真实学习场景。很多学生不是没有资料，而是面对一本厚教材时，不知道哪些内容是重点、哪些需要背、哪些容易混淆，也不知道怎么把教材内容转成可执行的复习任务。

所以我设计了考途 StudyAgent。它的核心目标是把教材复习链路产品化。用户输入考试类型、科目、章节标题、教材正文、当前基础、每日可用时间和复习周期后，系统会依次完成章节结构解析、重点知识提炼、背诵笔记生成、自测题生成、答案与解析生成和复习计划生成。

我选择 Dify Workflow 是因为这个任务不是一次普通问答，而是一个连续流程。把任务拆成多个节点后，每个节点的职责更清晰，也更容易测试和迭代。例如自测题节点只负责生成题目，答案解析节点严格对应原题，避免重新生成一套题。

在测试中，我也发现了一些问题，比如变量引用不正确会导致模型反问用户，多个 LLM 节点会消耗较多模型额度。所以后续我设计了变量检查、异常输入提示和省额度版本。下一步会接入考试大纲和历年真题知识库，让系统对考点判断更准确。

# 扫描版资料识别方案

## 1. 问题背景

部分考研资料书是扫描版 PDF，本质上是图片集合，而不是可复制文本。Dify 的文档提取节点更适合处理可复制文字的 PDF、Word、TXT 等文件。如果直接上传扫描版 PDF，可能出现以下问题：

- 无法提取正文
- 提取结果为空
- 识别顺序混乱
- 表格、页眉页脚、脚注被错误混入正文
- 公式、图表和批注识别不稳定

## 2. 产品处理策略

### MVP 阶段

MVP 阶段不直接处理整本扫描版教材，建议采用“用户预处理 + Agent 分析”的方式：

```text
扫描版 PDF
↓
OCR 工具识别为可复制文本
↓
用户按章节复制文本或上传识别后的 Word/PDF
↓
StudyAgent 生成章节学习报告
```

### 用户提示

当用户上传扫描版资料时，产品需要提示：

```text
如果你的 PDF 是扫描版图片，系统可能无法直接识别正文。建议先使用 OCR 工具将 PDF 转换为可复制文字，再按章节上传或粘贴正文。
```

## 3. 推荐输入方式

| 资料类型 | 推荐方式 |
|---|---|
| 可复制文字 PDF | 直接上传 |
| Word 文档 | 直接上传 |
| 扫描版 PDF | 先 OCR，再上传识别后的文本或 Word |
| 图片截图 | 先 OCR，再粘贴识别文本 |
| 整本书 | 优先按章节分段处理 |

## 4. 后续迭代方案

### 方案一：接入 OCR 预处理

在用户上传文件后，先通过 OCR 模块识别扫描 PDF，再把识别文本传入 Dify Workflow。

可选技术：

- PaddleOCR
- Tesseract OCR
- 云厂商 OCR API
- 带视觉能力的多模态模型

### 方案二：增加文件类型判断节点

在 Workflow 前置增加判断逻辑：

```text
用户上传文件
↓
判断是否可提取文本
↓
如果可提取：进入教材解析
↓
如果不可提取：提示用户先 OCR
```

### 方案三：支持按页 OCR

对于整本扫描教材，不建议一次性处理。可以按页或按章节切分：

```text
扫描 PDF → 按章节拆分 → OCR → 文本清洗 → Agent 分析
```

## 5. 简历与面试表达

可以在面试中这样说明：

```text
我在测试中发现，扫描版教材 PDF 并不能稳定被文档提取器识别，因为这类文件本质上是图片而不是文本。因此当前 MVP 阶段采用用户预处理方案，建议先 OCR 后再上传或粘贴文本。后续迭代会增加 OCR 预处理节点和文件类型判断机制，使系统能更稳定地处理扫描版资料。
```

# 迭代记录

## v0.1 原型搭建

- 使用 Dify Cloud 创建 Workflow 应用。
- 搭建用户输入、章节结构解析、重难点提炼、背诵笔记生成、自测题生成、复习计划生成、输出节点。

## v0.2 问题发现

- 发现变量未正确插入时，模型会反问用户提供章节内容。
- 发现存在两个“自测题生成”节点，节点职责需要区分。
- 发现多节点运行会消耗较多模型额度。

## v0.3 优化方向

- 将第二个自测题节点重命名为“答案与解析生成”。
- 增加考试类型、当前基础、复习周期等输入字段。
- 增加异常输入处理。
- 导出最新 DSL，并补充真实测试截图。

## v0.4 扫描版资料处理

- 发现部分考研资料书为扫描版 PDF，Dify 文档提取节点可能无法稳定识别。
- 当前 MVP 采用“先 OCR，再上传或粘贴文本”的处理方式。
- 后续计划增加 OCR 预处理节点和文件类型判断机制。

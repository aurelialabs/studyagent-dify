# 用户研究

## 目标用户

考研、考公、职业资格考试及大学课程考试中需要复习教材的学习者。

## 核心场景

用户正在复习一章教材，希望快速知道：

- 这一章讲了什么
- 哪些内容要背
- 哪些内容容易混淆
- 可能怎么考
- 今天应该怎么学

## 用户痛点假设

| 痛点 | 表现 |
|---|---|
| 重点识别难 | 教材内容多，难以判断考试重点 |
| 笔记整理慢 | 需要花大量时间摘抄、归纳、改写 |
| 输出不闭环 | 有笔记但没有自测题和复盘计划 |
| 计划不具体 | 只知道要复习，不知道每天做什么 |

## 当前验证方式

本项目目前通过原型测试验证功能链路，尚未进行大规模用户访谈。后续计划邀请 3-5 名备考用户试用，并收集输出质量、使用意愿和改进建议。
